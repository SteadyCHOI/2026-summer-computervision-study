## 2. 이미지의 자료 구조 이해

이미지는 추상적 그림이 아니라 **NumPy 배열**이며, 그 형태는 `(높이 H, 너비 W, 채널 C)`이다.

| 속성 | 의미 |
|------|------|
| `type(img)` | `numpy.ndarray` — 이미지는 곧 배열 |
| `img.shape` | `(H, W, 3)` — 세로·가로 픽셀 수와 채널 수 |
| `img.dtype` | `uint8` — 각 값은 0~255 정수 (0=어두움, 255=밝음) |

### 주요 확인 사항

- **채널의 정체**: 컬러 이미지의 마지막 차원 `3`이 채널(B, G, R)이다. 이는 이후 CNN에서 수십~수백 개로 확장되는 개념의 원형이다.
- **BGR 순서**: OpenCV는 RGB가 아닌 **BGR 순서**로 픽셀을 저장한다. 예) `[0, 0, 255]` = 빨강. 타 라이브러리와 혼용 시 색상 반전의 원인이 되므로 주의가 필요하다.
- **이미지 편집 = 배열 연산**: 픽셀 조작은 슬라이싱을 통한 값 대입으로 이루어진다.

```python
img[0:50, 0:50] = [0, 0, 255]   # 좌상단 50x50 영역을 빨강으로
```

### NumPy 인덱싱

이미지를 다루기 위해 NumPy의 다축 인덱싱을 정리하였다.

```python
arr[1, 2]        # 1행 2열 한 칸을 직접 지정 (정석)
arr[0:50, 0:50]  # 0~50행 AND 0~50열의 2차원 영역
```

`arr[1][2]`(축을 순차 접근)와 달리, 콤마 표기는 여러 차원을 동시에 지정한다. 특히 2차원 영역을 자를 때 `arr[0:50, 0:50]`과 `arr[0:50][0:50]`은 전혀 다른 결과를 내므로, 영역 추출에는 반드시 콤마 표기를 사용해야 한다.

### 색공간 변환

```python
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)   # shape: (H, W) — 채널 1
hsv  = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)    # shape: (H, W, 3)
```

흑백 변환 시 `shape`가 `(H, W, 3)`에서 `(H, W)`로 바뀌며 **채널이 3 → 1로 축소**됨을 확인하였다. 밝기 정보만 남기기 때문이다. HSV는 색상(Hue)·채도·명도로 분리되어 특정 색상 추출에 유리하며, 이후 차선 색상 필터링에 활용 가능하다.

## 3. 필터링 (Filtering)

필터링은 **커널(작은 숫자판)을 이미지 위에서 슬라이딩하며 곱하고 더해 새 값을 만드는 연산**이다. 이는 신호처리에서 다룬 1차원 컨볼루션의 2차원 확장에 해당한다.

### 적용한 필터

```python
blur   = cv2.GaussianBlur(img, (5, 5), 0)   # 종 모양 가중 평균
median = cv2.medianBlur(img, 5)             # 주변 픽셀의 중간값
```

- **Gaussian Blur**: 중심에 가중치를 더 주는 가우시안 분포 기반 평균. 자연스러운 흐림 효과.
- **Median Filter**: 주변 픽셀을 정렬해 중간값을 선택. 평균과 달리 이상치(점 노이즈)에 강건하여 소금-후추 노이즈 제거에 효과적.

### 커널 크기에 따른 관찰

커널 크기를 3×3 → 15×15 → 31×31로 키울수록 결과가 더 흐려졌다. **하나의 출력 픽셀을 계산할 때 평균에 참여하는 주변 픽셀 수가 증가하기 때문**이다(3×3=9개, 31×31=961개).

### 커널 직접 구현

```python
import numpy as np
kernel = np.ones((3, 3), np.float32) / 9    # 평균 필터
custom = cv2.filter2D(img, -1, kernel)
```

`filter2D`로 직접 정의한 평균 커널을 적용해 블러 효과를 재현함으로써, **블러 함수의 내부가 특정 커널의 컨볼루션**임을 검증하였다.

## 4. 엣지 검출 (Edge Detection)

엣지는 **밝기가 급격히 변하는 지점**이며, 이는 밝기의 변화량(미분의 이산 근사)으로 측정한다.

### Sobel — 변화량 측정 커널

```python
sobel_x = cv2.Sobel(gray, cv2.CV_64F, 1, 0, ksize=3)   # 가로 변화 → 세로선 검출
sobel_y = cv2.Sobel(gray, cv2.CV_64F, 0, 1, ksize=3)   # 세로 변화 → 가로선 검출
```

x방향 Sobel 커널의 구조:

```
-1  0  +1
-2  0  +2
-1  0  +1
```

- **음수와 양수의 조합**: 오른쪽(+)에서 왼쪽(−)을 빼는 구조로, "곱하고 더하는" 컨볼루션 틀에 음수 가중치를 넣어 **차분(변화량)**을 구현한다. 양수만으로 구성된 블러 커널이 누적·평활 효과를 내는 것과 대비된다.
- **중앙 행 가중치 2배**: 계산 대상 픽셀과 같은 높이의 좌우 변화를 가장 신뢰하기 때문이다. 위·아래 행을 함께 보는 것은 노이즈 완화를 위한 평활 역할이다.
- **미분과의 관계**: 이미지는 픽셀이 이산적이므로 극한 기반의 연속 미분이 불가능하다. 따라서 Sobel은 `(오른쪽 픽셀 − 왼쪽 픽셀)` 형태의 **유한 차분(미분의 이산 근사)**을 수행한다.

### Canny — 정제된 엣지 검출

```python
edges = cv2.Canny(gray, 100, 200)
```

Canny는 Sobel 기반 변화량에 두 개의 임계값(threshold)을 적용한다.

- 상위 임계값(200) 초과: 확실한 엣지로 채택
- 하위 임계값(100) 미만: 엣지 아님으로 제거
- 두 값 사이: 확실한 엣지와 **연결된 경우에만** 채택

이 연결성 판단 덕분에 끊긴 점 노이즈는 제거되고 이어진 윤곽선만 남는다.

### 임계값에 따른 관찰

| 설정 | 결과 |
|------|------|
| 낮은 임계값 (50, 100) | 엣지가 많이 검출됨 (노이즈·잔선 포함) |
| 높은 임계값 (200, 300) | 강한 경계만 검출됨 |

차선 인식에서는 이 trade-off 조정이 핵심이다. 너무 낮으면 도로 위 잡티를 선으로 오인하고, 너무 높으면 흐린 차선을 놓친다.

## 5. 영역 처리: 모폴로지 · 컨투어 · ROI

### 모폴로지 연산

이진 이미지의 흰색 영역을 다듬는 연산이다.

```python
kernel = np.ones((3, 3), np.uint8)
dilated = cv2.dilate(edges, kernel, iterations=1)   # 팽창: 흰 영역 확장
eroded  = cv2.erode(edges, kernel, iterations=1)    # 침식: 흰 영역 축소
```

- **팽창(dilate)**: 끊어진 엣지선 연결, 선 두껍게
- **침식(erode)**: 작은 점 노이즈 제거

### 컨투어 검출

```python
contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
cv2.drawContours(result, contours, -1, (0, 255, 0), 2)
```

흰색 영역의 외곽선을 좌표 목록으로 추출한다. 물체 개수 계산, 면적·형태 분석에 활용되며, 유즈케이스(예: 빈 주차 자리 탐지)의 기반 도구가 된다.

### ROI (관심 영역)

```python
h, w = img.shape[:2]
roi = img[h//2:h, 0:w]   # 하단 절반(도로 영역)만 추출
```

주행 영상에서 차선은 화면 하단 도로 영역에 위치하므로, ROI로 도로만 잘라내면 하늘·건물 등 불필요한 영역의 엣지를 배제할 수 있다. 차선 인식 파이프라인의 표준 전처리 단계이다.

## 6. 통합: 차선 인식 전처리 파이프라인

1주차에 학습한 연산을 하나의 함수로 통합하여, 차선 인식의 전처리 골격을 완성하였다.

```python
import cv2

def preprocess(path):
    img   = cv2.imread(path)
    gray  = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    blur  = cv2.GaussianBlur(gray, (5, 5), 0)
    edges = cv2.Canny(blur, 100, 200)
    h, w  = edges.shape[:2]
    roi   = edges[h//2:h, 0:w]      # 하단(도로) 영역만
    return roi
```

```
이미지 → Gray 변환 → Gaussian Blur → Canny 엣지 → ROI 추출
        → (3주차: Hough Transform 적용 시 차선 검출 완성)
```

이 함수는 3주차 차선 인식 파이프라인의 출발 코드로 직접 활용된다.

## 7. 핵심 정리

1. **이미지 = `(H, W, C)` 형태의 NumPy 배열**, 값은 0~255 정수. OpenCV는 BGR 순서.
2. **필터링 = 커널 컨볼루션**. 신호처리 컨볼루션의 2차원 확장이며, 커널 값이 연산의 성격을 결정한다.
   - 양수 커널 → 누적·평활 (블러)
   - 음수+양수 커널 → 차분·변화량 (엣지)
3. **엣지 = 밝기 변화량(유한 차분)**. Sobel은 방향별 변화량, Canny는 임계값 기반 정제.
4. **모폴로지·컨투어·ROI**로 검출 결과를 다듬고 필요한 영역만 추출.
5. 전체를 **하나의 전처리 함수로 통합**하여 차선 인식 파이프라인의 기반 마련.

### CNN으로의 연결

본 주차의 핵심 통찰은 **"고전 영상 처리는 사람이 커널을 직접 설계하고, CNN은 그 커널을 데이터로부터 학습한다"**는 점이다. Sobel 커널을 사람이 손으로 정의했듯, CNN의 합성곱 레이어는 동일한 컨볼루션 연산을 수행하되 커널 값(음수·양수 포함)을 학습으로 결정한다. 이 연결은 이후 딥러닝 단계 학습의 토대가 된다.
