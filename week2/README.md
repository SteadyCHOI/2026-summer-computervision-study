### 이번 주차의 핵심 전환

- 정해진 파라미터 사용 → **데이터에 맞는 파라미터 탐색(튜닝)**

### 학습 데이터
유튜브 주행 영상은 화질과 코덱 문제가 있어, **AI허브 공개 주행 데이터셋**의 프레임 시퀀스를 사용하였다. 이 데이터는 이미 프레임 단위로 분할되어 있어 별도의 영상 디코딩 없이 `glob` + `imread` 반복만으로 처리할 수 있었다. 데이터셋은 바운딩박스(bb)·폴리곤(pg)·주행가능영역(fs)·시맨틱(sl) 등 라벨 유형별로 구성되어 있으나, 2주차는 라벨을 사용하지 않고 원본 프레임 이미지만 활용하였다. (bb 라벨은 6주차 YOLO 단계에서 재사용 예정)

## 1. 데이터 로딩: 영상 = 프레임의 연속

영상 처리는 결국 **이미지 처리를 매 프레임 반복하는 것**이라는 점을 확인하였다. `VideoCapture`로 영상을 디코딩하는 대신, 이미 분할된 프레임을 순차적으로 읽어 동일하게 처리할 수 있다.

```python
import cv2
import glob

frame_paths = sorted(glob.glob("원천데이터/bb/*.jpg"))
for path in frame_paths:
    frame = cv2.imread(path)     # 1주차의 img 와 동일한 (H, W, 3) 배열
    # ... 1주차에서 익힌 처리를 그대로 적용
```

`sorted`로 파일명을 정렬하는 것이 중요하다. 파일명 순서가 곧 프레임 순서이므로, 정렬하지 않으면 재생 순서가 뒤섞인다.

## 2. Canny 파라미터 튜닝

1주차의 기본값 `Canny(100, 200)`을 실제 주행 프레임에 적용하면, 도로 질감·그림자·차량 윤곽 등 불필요한 엣지가 다수 포함되거나 흐린 차선이 검출되지 않았다. 데이터에 맞는 파라미터 탐색이 필요하였다.

### 트랙바를 이용한 실시간 튜닝

값을 코드에서 수정하고 재실행하는 방식은 비효율적이므로, 슬라이더(트랙바)로 임계값을 실시간 조절하며 최적값을 탐색하였다.

```python
cv2.createTrackbar("low",  "canny", 50,  255, nothing)
cv2.createTrackbar("high", "canny", 150, 255, nothing)

while True:
    low  = cv2.getTrackbarPos("low",  "canny")
    high = cv2.getTrackbarPos("high", "canny")
    edges = cv2.Canny(blur, low, high)
    cv2.imshow("canny", edges)
    if cv2.waitKey(1) == ord('q'):
        break
```

### Canny 이중 임계값의 원리 (히스테리시스)

Canny는 두 임계값으로 각 픽셀을 세 구간으로 분류한다.

| 조건 | 분류 | 처리 |
|------|------|------|
| 강도 > high | 확실한 엣지 | 무조건 채택 |
| low ≤ 강도 ≤ high | 애매한 엣지 | **확실한 엣지에 연결된 경우만** 채택 |
| 강도 < low | 노이즈 | 무조건 제거 |

핵심은 애매한 구간의 처리, 즉 **히스테리시스**이다. 애매한 픽셀이라도 확실한 엣지에 연결되어 있으면 살리고, 고립되어 있으면 버린다. 이 연결성 판단 덕분에 흐릿한 차선 부분이 이어지고 고립된 노이즈는 제거된다. 실무에서는 high를 low의 2~3배로 설정하는 것이 일반적이다.

### 관찰
- **low 상향**: 잡선 감소, 그러나 흐린 차선이 끊길 위험
- **high 상향**: 강한 엣지만 채택, 기준 엄격화
- **blur 강화**: 노이즈 제거에 유리하나 과도하면 차선까지 뭉개짐 (노이즈 제거 ↔ 차선 보존의 trade-off)

## 3. 엣지 검출 기법 비교

"왜 Canny를 사용하는가"를 근거 있게 뒷받침하기 위해 여러 엣지 검출 기법을 동일 프레임에 적용해 비교하였다.

```python
sobel_x = cv2.Sobel(blur, cv2.CV_64F, 1, 0, ksize=3)
sobel_y = cv2.Sobel(blur, cv2.CV_64F, 0, 1, ksize=3)
sobel_mag = cv2.magnitude(sobel_x, sobel_y)      # 방향 무관 전체 엣지 강도
laplacian = cv2.Laplacian(blur, cv2.CV_64F)      # 2차 미분
canny = cv2.Canny(blur, 80, 160)
```

`CV_64F`(실수, 음수 포함) 결과는 `cv2.convertScaleAbs`로 절댓값을 취해 uint8로 변환해야 음의 엣지까지 올바르게 시각화된다.

### 비교 결과

| 기법 | 특징 | 노이즈 | 차선 적합성 |
|------|------|--------|-------------|
| Sobel X | 세로선 위주 (차선에 유리) | 많음 | 중간 |
| Sobel Y | 가로선 위주 | 많음 | 낮음 |
| Sobel Magnitude | 모든 방향, 두꺼운 엣지 | 많음 | 중간 |
| Laplacian | 2차 미분, 매우 민감 | 아주 많음 | 낮음 |
| **Canny** | 얇고 깔끔한 이진 선, 연결성 처리 | 적음 | **높음** |

Sobel·Laplacian은 회색조의 엣지 강도를 출력해 추가 후처리가 필요한 반면, Canny는 히스테리시스를 통해 이미 정제된 이진 엣지를 제공하고 노이즈도 억제한다. 이 비교를 근거로 차선 인식 파이프라인의 엣지 검출 기법으로 Canny를 채택하였다.

## 4. ROI 설계: 순서와 형태

### 직사각형 ROI vs 사다리꼴 ROI

주행 영상에서 차선은 화면 하단 도로 영역에 위치하므로, ROI로 도로 영역만 남기면 하늘·건물의 불필요한 엣지를 배제할 수 있다. 두 형태를 검토하였다.

- **직사각형 ROI**: 배열 슬라이싱으로 이미지가 실제로 작아진다.
- **사다리꼴 ROI**: 원근을 고려해 도로 모양(하단은 넓고 소실점으로 갈수록 좁아짐)에 맞춰 마스킹한다.

```python
def apply_trapezoid_roi(edges):
    h, w = edges.shape[:2]
    mask = np.zeros_like(edges)
    polygon = np.array([[
        (int(w*0.10), h), (int(w*0.45), int(h*0.60)),
        (int(w*0.55), int(h*0.60)), (int(w*0.90), h),
    ]], np.int32)
    cv2.fillPoly(mask, polygon, 255)
    return cv2.bitwise_and(edges, mask)
```

### 처리 순서 최적화

ROI 적용 순서는 연산 효율과 정확성 모두에 영향을 준다.

- **직사각형 크롭은 Canny 이전에**: 이미지가 실제로 작아지므로 엣지 검출 연산량이 절감된다. 어차피 버릴 상단 영역을 미리 제거하는 것이 효율적이다.
- **사다리꼴 마스킹은 Canny 이후에**: 사다리꼴은 이미지 크기를 줄이지 못하고 바깥을 검정으로 채우는 방식이라, Canny 이전에 적용하면 **검정 영역과 도로의 경계에 인위적인 가짜 엣지**가 생긴다. 따라서 진짜 엣지를 검출한 뒤에 마스킹해야 한다.

| ROI 형태 | 적용 시점 | 이유 |
|----------|-----------|------|
| 직사각형 | Canny **이전** | 이미지 축소 → 연산 절감 |
| 사다리꼴 | Canny **이후** | 검정 경계의 가짜 엣지 방지 |

## 5. 조건별 강건성 테스트와 색 기반 보완

고정 파라미터가 다양한 주행 조건에서 어떻게 동작하는지 관찰하고, 엣지 검출의 한계를 보완하는 방법을 실험하였다.

### 조건별 한계

| 조건 | 순수 Canny 결과 |
|------|-----------------|
| 밝은 낮 | 양호 |
| 그림자 | 약함 — 그림자 경계가 차선으로 오인 |
| 야간/터널 | 약함 — 대비 부족으로 차선 소실 |
| 흐린 차선 | 약함 — 임계값 미만으로 검출 실패 |

### HSV 색 기반 차선 추출

엣지 검출은 밝기 변화에 의존하므로 그림자에 취약하다. 반면 차선의 **색(흰색·노란색)**은 밝기 변화에 상대적으로 강건하다. HSV 색공간은 색상(H)·채도(S)·명도(V)를 분리하므로 특정 색 추출에 유리하다.

```python
hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)
white  = cv2.inRange(hsv, (0, 0, 200), (180, 30, 255))    # 흰색: 채도 낮고 명도 높음
yellow = cv2.inRange(hsv, (15, 80, 100), (35, 255, 255))  # 노란색: 색상(H) 기반
lane_color = cv2.bitwise_or(white, yellow)
```

흰색은 "채도(S)가 낮고 명도(V)가 높다"는 특성을 이용해, 채도 상한과 명도 하한을 트랙바로 조절하며 최적 범위를 탐색하였다.

### 엣지 + 색 결합

엣지(형태)와 색(차선 색)을 결합하면 "차선 색이면서 엣지인 것"만 남아 강건성이 향상된다.

```python
color_dilated = cv2.dilate(color, np.ones((5,5), np.uint8), iterations=1)
combined = cv2.bitwise_and(edges, color_dilated)
```

얇은 엣지 선과 면 형태의 색 마스크가 잘 겹치도록, 색 마스크를 팽창(1주차 모폴로지 연산)시킨 후 결합하였다. 그림자 조건에서 순수 Canny 대비 안정적인 결과를 보였다.

## 6. 통합 전처리 파이프라인

Day 1~4의 요소를 하나의 함수로 통합하여 2주차의 최종 산출물을 완성하였다. 색 결합·ROI 적용 여부를 플래그로 조절할 수 있도록 설계하였다.

```python
def preprocess(frame, use_color=True, use_roi=True):
    h, w = frame.shape[:2]

    # 1) 직사각형 크롭 (연산 절감)
    top = h // 2
    cropped = frame[top:h, 0:w]

    # 2) gray → blur → Canny
    gray  = cv2.cvtColor(cropped, cv2.COLOR_BGR2GRAY)
    blur  = cv2.GaussianBlur(gray, (5, 5), 0)
    edges = cv2.Canny(blur, 80, 160)

    # 3) 색 결합 (선택)
    if use_color:
        color = cv2.dilate(get_color_mask(cropped),
                           np.ones((5,5), np.uint8), iterations=1)
        edges = cv2.bitwise_and(edges, color)

    # 4) 사다리꼴 ROI (선택, Canny 이후)
    if use_roi:
        edges = apply_trapezoid_roi(edges)

    # 원본 크기로 복원 (좌표 일관성 유지)
    full = np.zeros((h, w), np.uint8)
    full[top:h, 0:w] = edges
    return full
```

### 원본 크기 복원

크롭된 영역에서 처리한 엣지를 원본 크기 캔버스에 다시 얹어 반환한다. 3주차에서 검출한 차선을 원본 프레임에 그릴 때 좌표가 일치해야 하기 때문이다.

### 파이프라인 흐름

```
프레임 입력
  → 직사각형 크롭 (연산 절감)
  → gray → blur → Canny (튜닝된 값)
  → HSV 색 마스크 결합 (강건성)
  → 사다리꼴 ROI (가짜 엣지 방지)
  → 깨끗한 차선 엣지 출력  → [3주차 Hough Transform 입력]
```

## 7. 핵심 정리

1. **영상 처리 = 프레임 처리의 반복**. AI허브 프레임 시퀀스를 `glob`+`imread` 반복으로 처리.
2. **실제 데이터는 튜닝이 필수**. 트랙바로 Canny 임계값을 실시간 탐색.
3. **Canny의 강점은 히스테리시스**. 이중 임계값의 연결성 판단으로 정제된 이진 엣지 생성.
4. **기법 비교로 Canny 채택을 근거화**. Sobel·Laplacian 대비 노이즈·정제도에서 우위.
5. **ROI는 형태와 순서가 중요**. 직사각형은 Canny 전(효율), 사다리꼴은 Canny 후(가짜 엣지 방지).
6. **엣지의 한계를 색으로 보완**. HSV 색 기반 추출과 결합으로 그림자 조건 강건성 향상.
7. **전체를 통합 파이프라인으로**. 옵션 조절 가능한 `preprocess` 함수로 3주차 입력 준비 완료.

### 발견된 개선 과제 (4주차 고도화 대상)
- 그림자: 색 기반 보완이 효과적 → 결합 방식 정교화 필요
- 야간/터널: 대비 부족 → 히스토그램 평활화, 적응형 임계값 검토
- 흐린 차선: 근본적 한계 → 시간축 정보(프레임 간 연속성) 활용 여지
